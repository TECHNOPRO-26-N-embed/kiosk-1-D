### 3.1 画面遷移図

> テキスト形式で画面の状態遷移を記述してください。
> 例: `メインメニュー → [1] → 購入画面 → [確定] → 完了画面 → メインメニュー`
```
メインメニュー
 ├─ [1] 購入 → 商品スキャン画面
 │        ├─ 商品追加（バーコード入力 / 商品選択）
 │        ├─ カート確認
 │        └─ [確定] → 決済方法選択
 │                     ├─ 現金決済
 │                     │      └─ 入金処理 → 完了画面 → メインメニュー
 │                     └─ QRコード決済（将来拡張）
 │                            └─ 読み取り → 完了画面 → メインメニュー
 │
 └─ [0] 終了
```
### 3.2 データの洗い出し

> このシステムが**覚えておくべき情報**を洗い出してください。
> まずは「何の情報が必要か」を自然言語で書き出せばOKです。
> （C言語での型や構造体の定義は、明日の詳細設計で行います）

情報名|	      内容・説明	       例

商品ID:	商品を一意に識別する番号	100001
商品名:	商品の名称	おにぎり（鮭）
価格:	税抜または税込の販売価格	120円
税率区分:	店内/店外/免税などの税率種別	軽減税率（8%）
バーコード値:	商品を識別するバーコード番号	4901234567890  仮
カテゴリ:	商品分類（任意）	食品、飲料、雑貨                仮
商品在庫数:	在庫管理が必要な場合の数量	50個
税率ID:	税率設定を識別するID	T001
税率:	適用される税率	10%
適用条件:	店内/店外/免税などの条件	店外（持ち帰り）
取引ID:	会計処理ごとの識別番号	TRX20240501001
購入日時:	取引が行われた日時	2026/05/19 13:45
購入商品一覧:	カートに入った商品情報	おにぎり×2、飲料×1
合計金額:	税込の支払総額	350円
税額内訳:	税率ごとの税額	8%：20円、10%：10円
決済方法:	現金、QRコードなど	現金
決済ステータス:	成功/失敗などの状態	成功
投入金額:	現金決済時の投入額	500円
不足金額:	支払いに足りない金額	50円
QR読み取り結果:	QRコード決済の認証結果	認証成功
カート内商品リスト:	会計中の一時的な商品情報	商品ID 100001 × 2
現在の画面状態:	どの画面にいるかの状態管理	決済方法選択画面
操作ログ:	操作内容の記録	商品追加、決済完了
エラーログ:	エラー発生時の記録	入力ミス：未登録商品
