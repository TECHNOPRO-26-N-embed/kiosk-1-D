## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

1. メインメニューを表示する
2. ユーザーの入力を受け取る
3. 入力値に応じて処理を分岐する
   - 入力が "1" の場合：
       a. 商品スキャン画面へ遷移
       b. カート操作（追加・削除・確認）
       c. [確定] が選ばれたら決済方法選択へ
       d. 決済処理（現金 or QR）
       e. 完了画面を表示
   - 入力が "0" の場合：
       プログラムを終了する
   - 上記以外の場合：
       エラーメッセージを表示しメインメニューへ戻る
4. メインメニューに戻る（1へループ）

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

機能1: 購入処理
1. 商品スキャン画面を表示する
2. ユーザーに操作を選択させる（1:商品追加 / 2:カート確認 / 9:確定）
3. 入力に応じて処理分岐
   - 1の場合：バーコード入力または商品選択 → 商品をカートに追加 → 2へ戻る
   - 2の場合：カート内容を表示 → 2へ戻る
   - 9の場合：決済方法選択へ進む
   - その他：エラー表示 → 2へ戻る
4. 決済処理へ遷移

機能2: 商品追加処理
1. 商品入力方法を選択させる（1:バーコード入力 / 2:商品一覧から選択）
2. 入力に応じて処理分岐
   - 1の場合：バーコード値を入力 → 商品マスタ検索
   - 2の場合：商品一覧を表示 → 商品番号を選択
3. 商品が見つかった場合：カートに数量1で追加
4. 商品が見つからない場合：エラー表示
5. 商品スキャン画面へ戻る

機能3: カート確認処理
1. カート内の商品一覧を表示する
2. ユーザーに操作を選択させる（1:削除 / 2:数量変更 / 9:戻る）
3. 入力に応じて処理分岐
   - 1の場合：削除する商品IDを入力 → カートから削除
   - 2の場合：商品IDと新しい数量を入力 → 数量を更新
   - 9の場合：商品スキャン画面へ戻る
   - その他：エラー表示 → 1へ戻る
4. カート確認画面へ戻る

機能4: 決済方法選択
1. 決済方法メニューを表示（1:現金 / 2:QRコード / 9:戻る）
2. ユーザーの入力を受け取る
3. 入力に応じて処理分岐
   - 1の場合：現金決済処理へ
   - 2の場合：QR決済処理へ（拡張）
   - 9の場合：商品スキャン画面へ戻る
   - その他：エラー表示 → 1へ戻る

機能5: 現金決済処理
1. 合計金額を表示する
2. ユーザーに投入金額を入力させる
3. 投入金額が合計金額以上か判定
   - 以上の場合：お釣りを計算 → 完了画面へ
   - 未満の場合：不足金額を表示 → 2へ戻る
   
機能6: 完了画面表示
1. 「お買い上げありがとうございました」を表示
2. レシート情報（商品一覧・合計金額）を表示
3. メインメニューへ戻る

機能7: ログ保存処理
1. 取引IDを生成する
2. 購入日時を取得する
3. カート内容をまとめる（商品ID・数量・小計）
4. 税額内訳を計算する
5. 決済方法を記録する
6. CSV形式に整形する
7. 取引ログCSVに追記する
8. 必要に応じて明細CSVにも追記する

機能8: ログ表示処理　　
1. ログCSVを読み込む
2. 指定条件（期間・取引IDなど）があればフィルタする
3. ログ一覧を表示する
4. 詳細を見たい場合は取引IDを入力 → 明細を表示
5. メニューへ戻る

### 4.3 エラーハンドリング（想定される異常系）

> ユーザーが「想定外の操作」をした場合にどうするか検討してください。

| 異常系シナリオ | 対処方針 |

メニューに存在しない番号を入力	例：メニューが「1・0」なのに「5」などを入力	「無効な番号です」と表示し、再入力を促す
数字以外の文字を入力	例：「abc」「!」「空文字」	「数字を入力してください」と表示し、再入力
商品IDに存在しない番号を入力	商品マスタに該当なし	「商品が見つかりません」と表示し、商品入力に戻す
数量に0や負の値を入力	例：0、-1	「数量は1以上を入力してください」と表示し、再入力
カートが空なのに確定を押す	購入商品がない状態で確定	「カートが空です」と表示し、商品スキャン画面へ戻す
決済方法選択で存在しない番号を入力	例：メニューが「1・2・9」なのに「7」	「無効な選択です」と表示し、再入力
現金投入額が数値でない	例：「abc」「空文字」	「金額は数字で入力してください」と表示し、再入力
現金投入額が不足している	合計350円に対して100円など	不足金額を表示し、再入力
QRコード読み取り失敗	読み取り不可・通信エラー	「読み取りに失敗しました」と表示し、決済方法選択へ戻す
CSV保存に失敗	権限なし・ファイル破損	「ログ保存に失敗しました」と表示し、エラーログに記録
ログ読み込みに失敗	ファイルなし・フォーマット不正	「ログを読み込めません」と表示し、メニューへ戻す
予期しない例外が発生	プログラム内部エラー	try-catchで捕捉し、エラーメッセージ表示＋ログ記録＋メニューへ戻す
