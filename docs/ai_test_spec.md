# 🧩 セルフレジ精算機システム — 結合テスト仕様書
# 1. テスト目的
複数コンポーネント（商品選択、カート、決済、税計算、CSV出力、レシート出力）が
相互に連携して正しく動作すること を確認する。

特に以下を重点確認する：

画面遷移の整合性

カート内容と決済金額の整合性

税率計算と決済処理の連携

決済完了後の CSV 出力・レシート出力の一貫性

エラー発生時の全体的な復旧動作
------
# 2. テスト範囲
メインメニュー → 商品一覧 → 商品選択 → カート → 決済 → CSV出力 → レシート

税率計算と金額計算の連携

取引ID生成とログ出力

異常系の画面遷移・復旧
------
# 3. テスト観点
機能結合（画面遷移・データ受け渡し）

金額整合性（カート → 税計算 → 決済）

CSV整合性（決済結果とログ内容の一致）

異常系結合（途中エラー時の復旧）

境界値（数量・金額・税率）
